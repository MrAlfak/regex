# مشارکت در API-Security-Checklist

1. **Fork** و سپس یک Branch با نام توصیفی بسازید:  
   ```bash
   git checkout -b feature/add-rate-limit-section
   ```
2. تغییرات خود را با توضیح شفاف Commit کنید.
3. تست‌ها یا مثال‌های مرتبط با مورد جدید را (در صورت وجود) به‌روزرسانی کنید.
4. یک Pull Request باز کنید و در توضیحات، مشکل یا انگیزه اصلی تغییر را ذکر کنید.

## راهنمای سبک
- متن اصلی فارسی است؛ اگر منبع انگلیسی می‌آورید، لینک بدهید.
- برای آیتم‌های چک‌لیست از `- [ ]` استفاده کنید تا Markdown به درستی رندر شود.
- در صورت افزودن منبع جدید، آن را در بخش «منابع پیشنهادی بیشتر» اضافه کنید.

## کد رفتاری
از تعامل محترمانه و پاسخ‌گو بودن استقبال می‌کنیم؛ هرگونه رفتار آزاردهنده گزارش و پیگیری می‌شود.
